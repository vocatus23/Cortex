# Simplicity first

Always take the simplest route to solve a problem. Avoid circular complexity,
over-engineering, and premature abstraction.

**Why:** Multiple incidents where complex solutions introduced more bugs than
the original problem. The simplest fix is usually the most reliable.

**How to apply:** Before implementing, ask: "Is there a simpler way?" If yes,
do that first. Only add complexity when the simple approach has been proven
insufficient with real data.
